双脑架构

每一条 AI 工具调用都经过本地小脑的语义安全审查。敏感信息永不出设备，审查结果分为 approve / flag / reject 三级， 附带风险等级和审查原因。智能跳过低风险只读操作，不影响性能。

用户请求 → 大脑推理 → tool_call → 小脑审查 → 执行

零命令行

全部操作通过图形界面完成。内嵌 Next.js Web 前端，以单个可执行文件分发，开箱即用。

设计原则

零依赖核心

internal/core 只依赖标准库，是整个系统的类型基石。

领域隔离

domain 包之间不互相依赖，全部通过 core 共享类型通信。

Runtime 中枢

runtime.Runtime 聚合配置、会话、任务、投递、渠道、工具和本地模型能力，是 API 与执行主链路共享的系统门面。

统一执行管线

WebChat、外部渠道、任务、心跳与子代理最终都复用同一条 6 阶段执行管线，而不是各自维护独立执行器。

单二进制交付

前端构建产物会同步到 api/static/dist 并被 Go embed 打包，最终以一个主程序同时提供页面与 API。

多层安全防线

纵深防御，每一层都是一道屏障

L1

工具策略

allow / deny / profile / group，子代理深度限制

L2

提权门控

高危操作分级拦截（low → critical），需用户确认

L3

沙箱隔离

文件操作限制在授权目录，系统关键路径只读

L4

工具审批

敏感工具需通过动态审批流程

L5

循环检测

重复 / 乒乓调用自动熔断，全局熔断器保护

L6

小脑语义审查

本地模型对 tool_call 做意图一致性、注入攻击、数据外泄等语义分析

L7

会话隔离

会话间访问控制，4 级可见性策略（self / tree / agent / all）

Terminal 
./kocort

启动后访问本地页面即可使用 WebChat 与管理面板；如需云端模型，再到设置中补充 provider 配置。

本地模型

或
